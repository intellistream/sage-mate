- **Policy-Driven Workflow Engine**: The core `service.py` executes a linear or DAG-based pipeline where a `DeterministicWorkflowPlanner` generates a `PlanSpec` based on `workflow_policy.json`. A `WorkflowPolicyValidator` enforces constraints (latency budgets, evidence contracts, side-effect permissions) before execution.
- **Plugin Architecture**: `CapabilityPluginRegistry` loads JSON manifests from `data/capability_plugins/`, merging custom `WorkflowStepDefinition`s into the global registry. The planner dynamically injects plugin steps (e.g., `retrieve_research_overview`) based on intent classification in `workflow_planner.py`.
- **Runtime Bootstrap & Isolation**: `runtime_env.py` and `tools/lib/runtime_env.sh` enforce a deterministic `PYTHONPATH` that prioritizes local sibling checkouts (`../SAGE`, `../sageVDB`, `../neuromem`) over PyPI installs, auto-linking compiled `.so` extensions for `sageVDB` to ensure development/production parity.
- **Shared State & Telemetry**: All modules share a unified `AppSettings` (Pydantic) for configuration. Workflow execution emits structured `WorkflowTraceStep`s via a `WorkflowEventBroker` (SSE) for real-time UI updates, while side-effects (memory writes, booking drafts) are gated by `draft_write` permissions and stored in file-based JSON stores.
