- **Entry Point & API Layer**: `api.py` defines a FastAPI application (`llm_app`) exposing REST endpoints for chat, authentication, knowledge management, and administrative operations. It utilizes a `LazyDigitalTwinService` proxy to defer heavy initialization and a `WorkflowEventBroker` for Server-Sent Events (SSE) streaming of workflow traces.
- **Core Service Logic**: `service.py` implements the `DigitalTwinService`, orchestrating the end-to-end chat pipeline. It integrates a `DeterministicWorkflowPlanner` (`workflow_planner.py`) to generate execution plans based on user intent and policy, and executes them via a series of stages (e.g., `retrieve_knowledge`, `llm_answer`, `memory_persist`).
- **LLM Integration**: `llm_client.py` provides `VllmChatClient`, a specialized HTTP client for vLLM-HUST that supports streaming, semantic caching, intent classification, and DeltaKV session continuity hints for KV-cache reuse.
- **Data Persistence**: The module employs a file-based storage architecture with dedicated store classes (e.g., `knowledge_base.py`, `memory_store.py`, `user_store.py`) managing JSON/SQLite backends for knowledge documents, conversation memory, user accounts, and operational queues.
- **Frontend**: A static single-page application (`web/app.js`, `web/index.html`) is served directly by the FastAPI app, handling UI state, SSE consumption, and administrative dashboards.
