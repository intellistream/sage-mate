- **Runtime Orchestration**: Uses `manage.sh` and `quickstart.sh` as the primary entry points to coordinate the lifecycle of five distinct systemd user services: the vLLM engine, OpenAI-compatible proxy, main FastAPI app, local site proxy, and Cloudflare tunnel.
- **Service Wiring**: The `quickstart.sh` script renders systemd unit templates from `deploy/systemd/user/`, injecting repository paths and Python binary locations to enforce isolation and correct environment loading for each child module.
- **Shared State Contract**: All child modules (app, tests, deployment) rely on a unified file-based state directory (`data/`) for conversation memory, user accounts, and domain knowledge, ensuring consistency across restarts and service boundaries.
- **Dependency Management**: Leverages a monorepo-style installation via `quickstart.sh` that optionally clones and installs sibling repositories (SAGE, neuromem, vllm-hust) into a shared virtual environment, establishing a common tech foundation for the app and inference engine.
