- **Runtime Bootstrapping**: Uses `runtime_env.py` to dynamically prepend sibling repository paths (SAGE, sageVDB, neuromem) to `sys.path` and auto-link compiled `.so` extensions, ensuring a unified development environment without manual PYTHONPATH configuration.
- **Workflow Orchestration**: Implements a 13-stage linear or DAG-based chat pipeline (`service.py`) using the SAGE FlowNet runtime. Stages include intent understanding, booking preparation, hybrid knowledge/memory retrieval, prompt assembly, LLM answer generation, and post-answer memory persistence.
- **Policy Enforcement**: Enforces cross-cutting constraints via `WorkflowPolicyValidator`, which validates deterministic plans against role-based access controls, evidence source consent, and latency budgets before execution.
- **Shadow Planning**: Runs a parallel LLM-based "shadow planner" alongside the deterministic planner to generate alternative workflow candidates for offline comparison and continuous policy refinement.
- **Lazy Service Initialization**: Uses a `LazyDigitalTwinService` proxy in the FastAPI layer to defer heavy resource initialization (LLM clients, vector stores) until the first request, optimizing cold-start performance.
