- **Entry Point & API**: `src/sage_faculty_twin/api.py` exposes a FastAPI application (`llm_app`) with endpoints for chat, authentication, knowledge management, and administrative operations. It uses a `LazyDigitalTwinService` to defer heavy initialization.
- **Core Orchestration**: `src/sage_faculty_twin/service.py` implements the `DigitalTwinService`, which orchestrates the chat workflow through a series of stages (intent understanding, booking preparation, knowledge/memory retrieval, prompt building, LLM interaction, and post-answer persistence).
- **Workflow Planning**: `src/sage_faculty_twin/workflow_planner.py` contains the `DeterministicWorkflowPlanner`, which generates execution plans based on user intent and policy constraints, supporting both deterministic and shadow-planning modes.
- **LLM Interaction**: `src/sage_faculty_twin/llm_client.py` provides the `VllmChatClient` for communicating with vLLM-based inference engines, featuring streaming support, semantic caching, and DeltaKV session continuity hints.
- **Configuration**: `src/sage_faculty_twin/config.py` uses `pydantic-settings` to manage environment variables prefixed with `DIGITAL_TWIN_`, controlling model endpoints, retrieval settings, and workflow policies.
- **Data Persistence**: The module relies on a file-based storage system under `data/` for conversation memory, knowledge bases, user accounts, and operational state, managed by various store classes (e.g., `MemoryStore`, `KnowledgeBase`).
